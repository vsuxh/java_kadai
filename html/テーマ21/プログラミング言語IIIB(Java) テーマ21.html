jarファイル

Javaで作成したソフトウェア(アプリ)を配布する方法の一つとして, jarファイルがある. jarファイルを用いると, プログラムのクラスファイルやアニメーションに用いる画像ファイルなど, アプリの実行に必要な多数のファイルを, 一つのアーカイブファイルにまとめることができる. 単一のファイルになって取扱いが容易になる上に, ファイル圧縮が行われるためデバイス間やネットワーク等での転送が高速になるという利点もある.


図: 複数のファイルをまとめてjarファイル化し, jarファイルのみを配布する.

jarファイルの作成にはjarコマンドを用いる. jarコマンドの使い方は, UNIX系OS(Linux, Mac, WSL等)の標準アーカイブコマンドであるtarコマンドと同様であるため, UNIXユーザーには馴染みが深く, 取り掛かりが容易である.

*.class と *.png をまとめて sample1.jar を作成(create)するには

          ↓ここにjarファイルを指定
$ jar vcf sample1.jar *.class *.png
      ↑命令          ↑最後にjarファイルに入れたいファイルを列挙

※命令「vcf」の中で最も重要なのはcである. 「createのc」と暗記せよ.
※なお, vはverbose, fはfileである. fileはjarファイルを指定することを意味する.
　verboseは冗舌の意味で, ログをたくさん出力してくれる.

sample1.jar に含まれるファイルの一覧(list)を表示するには
          ↓ここにjarファイルを指定
$ jar vtf sample1.jar
      ↑命令 エルではなくティー
※「listのt」と暗記せよ.

マニフェストファイルによるmainの指定

さらにjarファイルにマニフェストファイルも追加すると, 実行するmainメソッドがあるクラスを指定できる. ※「マニフェスト」は「声明文」や「選挙公約」という意味だが, ここではあまり気にしない方が良い.

事前にマニフェストファイルを manifest.txt などの名前で作成しておく.
その内容は1行のみで,

Main-Class: mainメソッドがあるクラスの名前 MyFrame など

jarファイルを作成する際にマニフェストファイルも指定する.
        ↓ここにmを追加
$ jar vcfm sample1.jar manifest.txt *.class *.png
                       ↑ここにマニフェストファイルを指定

こうすることで, jarファイルのまま実行できる様になる.
$ java -jar sample1.jar
sample1.jarの中の, マニフェストファイルで指定したmainメソッドからプログラムが実行される.

画像ファイルの読込みのjarファイル対応方法

jarファイル化した際に, カレントディレクトリではなくjarファイルの中から画像ファイルを読み込む様に, URLクラスを使う形式にコード修正しておく.

プログラム冒頭で
import java.net.*;

コンストラクタで
Toolkit tk = Toolkit.getDefaultToolkit();
URL url = this.getClass().getResource(画像ファイル名); // この行を入れてURL方式にし
Image chara = tk.getImage(url);                        // tk.getImageの引数をURLにする

画像ファイルが多くなるとディレクトリで分類したくなると思うが, ディレクトリ指定を含めると事情が複雑になり余計な不具合を抱える可能性が高まるため, 今回はディレクトリを使わずにフラットな配置にして欲しい.

ヘルプとアドバイス

  • うまく動かない場合のチェックポイント:
    • そもそもjavacでコンパイルしたか?
    • 動作に必要なファイルが全てjarファイルの中に入っているか?
  • デバッグの基本はデバッグプリントである. プログラム中の随所にSystem.out.printlnを挿入し, その行を通過していることや, 変数の値が期待通りになっていることを確認せよ. 端末ではなく描画パネルに変数の値を随時表示(描画)しても良い. 
    System.out.println("x=" + x + " y=" + y); // 起動した端末へ表示
g.drawString("x=" + x + " y=" + y, 16, 16); // 描画パネル上に表示(文字列をグラフィックスとして描画)

課題

今回はレポート提出システムによるアップロード方式ではなく, 学内ファイルサーバーの学内ウェブ公開用ディレクトリに jar ファイルや html ファイルをコピーすることで提出とする.

開発中は従来通り, 今回のテーマの課題遂行用ディレクトリで作業し, そこでjarファイル化して動作確認を行った後, 公開用ディレクトリへコピーすると良い. ファイル名やディレクトリ名に不備があると自動回収が不可能となり, 未提出として扱われるため, ディレクトリ名やファイル名が指定された通りになっているか, 良く確認すること.

【課題】前回のテーマで作成したアニメーションのプログラムをjarファイル化して公開せよ. 以下の条件を満たす様に作成すること.

  • 公開用ディレクトリは各自の ~/public_html/java/app21 とする.
  • jarファイルのファイル名は「ユーザーID.jar」とする. ※今年度の皆さんのユーザーIDはおおむね「p22xxxx」
  • HTMLファイルの名前は ユーザーID.html とする.
    • ページの最初に出席番号・名前・テーマ番号を記入する.
    • 自動回収の都合上, 複数のHTMLファイルには分割せず, ユーザーID.html ひとつで作成して欲しい.
    • CSSやJavascriptを使用して装飾しても良いが, 別ファイルになっていると自動回収は保証できないので, 埋め込みを推奨する. (別ファイルかつ学外のファイルは論外. 学外のファイルを指定しないこと.)
  • HTMLファイルからソースファイルを閲覧できるようにする. ソースファイルを ユーザーID.html と同じ場所に置き, ユーザーID.html からリンクする. ※どうしても非公開にしたい場合は教員に相談・確認の上, 直接提出すること.
  • ディレクトリ構成は以下の通り.
    • ~/public_html/java/app21/ の直下に
         ユーザーID.html  (ここから各ファイルへリンクする)
         ユーザーID.jar   (この中にマニフェストファイルとクラスファイルと画像ファイルを入れる)
         MyFrame.java     (閲覧用) ソースファイル名はこの通りでなくても良い
         MyPanel.java     (閲覧用) ソースファイル名はこの通りでなくても良い
  • ユーザーID.html に説明を記入する. スクリーンショットが入るとなお良い.
    • ユーザー向けの説明 ... アニメーションの内容や, (もし操作があれば)操作方法の説明
    • 開発者向けの説明 ... アニメーションの仕組みの説明. どの様に座標等の変数を変化させているか.
  • [注意] 成績保管のため, 今回作成したウェブページは消去せずに残しておくこと.
  • 主な評価項目
    1. 所定のディレクトリに配置されており, 他の人がダウンロードして動作させることが可能であるかどうか
    2. 二つの説明文(ユーザー向け, 開発者向け)
    3. jarファイルやソースファイルへのリンク
