内容

Javaで配列を扱う方法を学ぶ.

C言語と同様に, メモリブロックの「アドレス」という考え方は登場するものの, &や*, ++, --等のポインタ演算ができない安全な仕様であるため, C言語の配列よりも理解や利用が容易なので安心して欲しい.

基本型の配列

  • 基本型とは, intやdouble等の値を「一つ」格納できる変数の型である.
  • 配列はメモリブロックであり, インスタンスと同様にnew演算子で生成する. new演算子はメモリブロックのアドレスを返すので, それを配列型の変数(配列変数)に保存しておき, [ ]演算子で各要素にアクセスする.
  • new演算子で配列を生成する際は, 要素数(要素の個数)を指定する.

    • ※配列の「要素」「要素の番号」「要素数」「配列変数」をしっかり区別すること!

    ※ひとつひとつの「要素」のことを, 「配列」と呼ばないこと!

    ※【大切な事なのでもう一度】 ひとつひとつの「要素」のことを, 「配列」と呼ばないこと!!

    // 例: int型10個の要素から成る配列

int[] a;         // int型の配列変数aを宣言. この段階では矢印の根本のみ. まだメモリブロックはない.
a = new int[10]; // int型10個分のメモリブロックを割り当て, そのアドレスをaに代入.
                 // これによりaの矢印がメモリブロックを「指す」ようにつながる.

a[0] = 314;      // [ ]演算子で各要素にアクセスできる. 0番から始まる.
a[1] = 271;
...
a[9] = 256;      // int型10個分なので, 使えるのは9番まで.
                 // もし10番にアクセスすると実行時エラーとなり, ArrayIndexOutOfBoundsExceptionが出る.
                 // (要素番号が, 使える範囲から外れていますよ, の意)
  • 配列の要素数は, 配列変数のlengthフィールドで取得できる(フィールドなので括弧が付かない).
    • int n = a.length; // nに, 配列aの要素数が代入される. lengthに()は付けない.

オブジェクトの配列

  • 基本型の配列では要素に値がそのまま格納されるが, オブジェクトの配列では, 各要素にはインスタンスへの参照(アドレス)が格納される.

    • ※つまり「オブジェクト変数の配列」と理解するのが正しい.

  • 各要素には, 配列の生成時に指定したクラスの参照(アドレス)のみならず, そのクラスを継承したクラスの参照(アドレス)も代入できる. →課題ではこれを利用して通学生と寮生を一緒に入れた「高専生の」配列を作る.
    • // 例: 曲データの配列

Song        song_a = new Song(〜);
Song        song_b = new Song(〜);
Song        song_c = new Song(〜);
KaraokeSong song_d = new KaraokeSong(〜); // この曲データのみ歌詞の文字列付き

Song[] a;         // Song型の配列変数aを宣言. 
a = new Song[10]; // 参照(アドレス)を10個格納できるメモリブロックを割り当てて, aから接続する.

a[0] = song_a; // a[0]の箱からsong_aにつながる.
a[1] = song_b; // a[1]の箱からsong_bにつながる. 以下同様.
a[2] = song_c; // aの要素にSong型を格納できるのは当り前だが,
a[3] = song_d; // KaraokeSong IS-A Songなので, KaraokeSong型も格納できる. これも継承の利点の一つ.

      • 「実際の要素数」のゲッター size ... 「実際の要素数」をreturnするだけ.
      • 要素のゲッター get ... 引数として要素の番号を受け取り, その番号の要素を返す. ※要素「数」のゲッターではない.
