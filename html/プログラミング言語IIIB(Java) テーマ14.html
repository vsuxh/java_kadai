JLabelクラス

文字列等, 単に説明やメッセージを表示するためのGUI部品で, 入力を行うことはできない. コンストラクタやメソッドはJButtonクラスとほぼ同様である.

// 自作プログラムのフィールドで
private JLabel label1;

// 自作プログラムのコンストラクタで
this.label1 = new JLabel("ダウンロード中(0%)..."); // コンストラクタの引数で文字列を与える
                                                   // あとから .setText( ) で変更も可能
super.getContentPane().add(this.label1);

JPanelクラスとレイアウト・マネージャ

JFrameクラスのみでは複数の部品を配置することができない(図1). nullレイアウトを使えば部品毎に座標やサイズを指定して配置することもできるが, 部品数が多くなると手間が掛かり過ぎて現実的ではない. また, nullレイアウトではウィンドウの大きさの変化や解像度(ピクセル数)の違いに対応できない融通の効かないアプリとなる.

そこでSwingでは, 複数の部品を自動配置するための機能として, JPanelクラスとレイアウトマネージャが用意されている(図2).


図1: 前回までの状態. 部品を一つしか配置できない.


図2: パネルを用いて複数の部品を配置した状態.

JPanelは複数の部品を一つにまとめる役割を果たす. JPanelのインスタンスに対し, まとめたい部品を次々とaddすることで, パネルに複数の部品を追加できる. そして最後に, そのパネルをウィンドウの枠にはめる(図3). 以下に例を示す.

// 自作プログラムのコンストラクタで
JPanel panel1 = new JPanel(); // JPanelはローカル変数で良い
panel1.add(this.button1); // パネルに対してボタン1を追加
panel1.add(this.button2); // パネルに対してボタン2を追加
panel1.add(this.button3); // パネルに対してボタン3を追加

super.getContentPane().add(panel1); // contentPaneに対してパネルを追加
// なお, contentPaneは, JFrameで予め用意されているウィンドウの「枠」


図3: 点線枠がcontentPane

パネルに部品を追加した際に部品が配置される位置は, レイアウトマネージャが自動的に判断する. レイアウトマネージャは部品を自動的に配置するクラスで, 配置の方針が異なる様々な種類が用意されている. レイアウトマネージャを設定するにはパネルのsetLayoutメソッドを用いる. 以下に例を示す.

// 自作プログラムのコンストラクタで
FlowLayout flow1 = new FlowLayout(); // レイアウトマネージャはローカル変数で良い
panel1.setLayout(flow1);

以下に, 配置の方針が異なる2種類のレイアウトマネージャを紹介する.

FlowLayoutクラス

部品をaddした順番に, 左から右へ並べる最も基本的なレイアウトマネージャである. パネルの横幅が部品すべてを横1行に並べるのに足りない場合は, 必要に応じて自動的に改行される.

// 自作プログラムのコンストラクタで
FlowLayout flow1 = new FlowLayout(); // レイアウトマネージャはローカル変数で良い
panel1.setLayout(flow1); // panelに部品をaddする「前」にこれが必要

panel1.add(this.label1);
panel1.add(this.button1);
panel1.add(this.button2);


BorderLayoutクラス

部品を「東・西・南・北・中央」で指定して配置する. 東(EAST)が右, 西(WEST)が左, 南(SOUTH)が下, 北(NORTH)が上, 中央はCENTERである(図4).


図4: ボーダーレイアウトの配置位置

部品の配置位置は, addメソッドの第2引数で指定する. 特に何も指定しないと中央となる. なお, 同じ位置に何度もaddすると上書きして配置されるため, 先にaddした部品が隠れてしまうので注意が必要である.

// 自作プログラムのコンストラクタで
BorderLayout border1 = new BorderLayout(); // レイアウトマネージャはローカル変数で良い
panel1.setLayout(border1); // panelに部品をaddする「前」にこれが必要

panel1.add(this.button1, BorderLayout.WEST); // ボタン1を左(WEST)に配置
panel1.add(this.button2, BorderLayout.EAST); // ボタン2を右(EAST)に配置
panel1.add(this.label1, BorderLayout.SOUTH); // ラベル1を下(SOUTH)に配置

JPanel及びレイアウトマネージャの使用上の注意

  • パネルのインスタンスはローカル変数(その場限りの使い捨て)で良い.
  • レイアウトマネージャのインスタンスも同様にローカル変数で良い.
  • レイアウトマネージャの設定はパネルに部品を追加(add)する前に行う. addの後に設定しても意味がない.
    • // 自作プログラムのコンストラクタで
panel1.setLayout(flow1); // これが先
                         // 部品配置の方針を先に決めてから

panel1.add(this.label1); // 部品をパネルに追加する
panel1.add(this.button1); // するとレイアウトマネージャが自動的に部品を配置してくれる
panel1.add(this.button2);

super.getContentPane().add(panel1); // 最後にパネルをcontentPaneに追加する
  • [厳重注意] レイアウトマネージャのインスタンスの使い回しは出来ない. エラーとはならないが, 正常動作しなくなる. 複数のパネルがある場合はレイアウトマネージャのインスタンスもそれぞれ用意する必要がある.
    • panel1.setLayout(flow1);
panel2.setLayout(flow1); // ダメな例. flow1を使い回ししている.
~~~~~~           ~~~~~
  • [厳重注意] 部品の配置がどれほど複雑になっても, contentPaneに対するaddは1回に限る. 2回以上addすると正常動作しなくなる.
    • super.getContentPane().add(panel1);
super.getContentPane().add(panel2); // 2回以上のadd禁止. panel1が隠れて消えてしまう.
  • パネル自身も部品であるから, パネルの中に他のパネルを入れることが可能である. これをパネルの入れ子と呼ぶ.
    • panel2.add(this.button1);
panel2.add(this.button2); // パネルの2番にボタンを2つ入れておいて

panel1.add(panel2); // ここに注目
panel1.add(this.label1); // パネルの1番に, パネルの2番とラベルを入れる

super.getContentPane().add(panel1);
  • 前回テーマ13で試した「nullレイアウト」は, レイアウトマネージャを「無効」 にする事を意味する. これにより, 手間は掛かるが完全に自由に部品を配置することはできる. しかし, 部品のサイズと位置を固定するため, ウィンドウのサイズ変更やデバイスの解像度(ピクセル数)のバリエーションに対応できない融通の効かないアプリとなってしまう.
